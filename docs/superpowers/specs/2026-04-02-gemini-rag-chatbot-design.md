# Gemini RAG Chatbot for Usage Report

**Date:** 2026-04-02
**Status:** Design
**Approach:** A — Next.js API Route + Client-side Chat Bar

## Summary

Add an AI chatbot to the usage-report app that uses Gemini File Search (RAG) to answer leadership questions about Cursor usage data, commits, PRs, and the report's claims. Uses `@google/genai` SDK with a File Search Store for grounded, citation-backed answers.

## Audience

Management/leadership reviewing the report. Tone: professional, clear, non-technical. The AI should explain data in business terms and support questions like "justify this cost" or "what was delivered."

## Architecture

```
[Browser] <ChatBar />
    |
    | POST /api/chat  (SSE stream)
    v
[Next.js API Route]  src/app/api/chat/route.ts
    |
    | generateContentStream() with fileSearch tool
    v
[Gemini API + File Search Store]
    |
    | Retrieves relevant chunks from uploaded data
    v
[Response streamed back to browser]
```

## 1. Data Ingestion — Setup Script

**File:** `scripts/setup-rag.ts`

**Purpose:** One-time script to create a Gemini File Search Store and upload all report data.

**Steps:**
1. Initialize `GoogleGenAI` with `GEMINI_API_KEY` from environment
2. Create a File Search Store with display name `"usage-report-rag"`
3. Upload 6 files with custom metadata:
   - `public/data/cursor-usage.csv` — `{type: "cursor-usage"}`
   - `public/data/commits.csv` — `{type: "commits"}`
   - `public/data/prs.json` — `{type: "pull-requests"}`
   - `public/data/company-contributions.json` — `{type: "contributions"}`
   - `public/data/summary.json` — `{type: "summary"}`
   - `public/data/report-context.md` — `{type: "report-narrative"}`
4. Poll each upload operation until `done === true`
5. Write the store name to `.env.local` as `GEMINI_FILE_SEARCH_STORE`
6. Print success message with store name

**Idempotency:** If `GEMINI_FILE_SEARCH_STORE` already exists in `.env.local`, print a warning and exit unless `--force` flag is passed.

**Run command:** `npx tsx scripts/setup-rag.ts`

**Dependencies:** `@google/genai`, `tsx` (dev)

## 3. Chat API Endpoint

**File:** `src/app/api/chat/route.ts`

**Method:** POST

**Request body:**
```typescript
{
  message: string;
  history?: { role: "user" | "model"; content: string }[];
}
```

**Behavior:**
1. Validate request body (message required, non-empty)
2. Read `GEMINI_API_KEY` and `GEMINI_FILE_SEARCH_STORE` from env
3. Initialize `GoogleGenAI`
4. Build contents array from history + new message
5. Prepend system instruction:
   > "You are an AI assistant for Sanjay Kumar's Cursor Usage & Work Activity Report. You help leadership understand the data. Answer professionally, cite specific numbers and dates from the data. If you don't know something, say so. Do not make up data."
6. Call `ai.models.generateContentStream()` with:
   - Model: `"gemini-2.5-flash"` (confirmed to support File Search per docs)
   - Tools: `[{ fileSearch: { fileSearchStoreNames: [storeName] } }]`
   - The constructed contents
7. Stream response chunks as SSE (`text/event-stream`)
8. After stream completes, send grounding metadata (citations) as a final SSE event

**Response format (SSE):**
```
data: {"type": "chunk", "text": "The highest..."}
data: {"type": "chunk", "text": " token usage day was..."}
data: {"type": "done", "citations": [...]}
```

**Error handling:**
- Missing env vars: 500 with `{"error": "Server not configured"}`
- Gemini API error: 502 with `{"error": "AI service unavailable"}`
- Invalid request: 400 with `{"error": "Message is required"}`

## 4. Chat UI Component

**File:** `src/components/chat-bar.tsx` (client component)

### Collapsed State
- Fixed to bottom of viewport, full width, z-index above content
- Slim bar (~56px height) with:
  - Left: sparkle/chat icon + "Ask about this report..."
  - Right: send button (purple accent)
- Typing in the input auto-expands to the expanded state
- Subtle glass/blur background matching the report's dark theme

### Expanded State
- Max height: 60vh
- Smooth transition from collapsed
- Layout (top to bottom):
  - Header bar: "Report Assistant" label + collapse chevron button
  - Message area (scrollable):
    - User messages: right-aligned, subtle background
    - AI messages: left-aligned, card-style with slight purple border
    - Citations: small Badge components below AI messages showing source file names
  - Input area: text input + send button (same as collapsed)

### Streaming Behavior
- On send: add user message to history, show AI message bubble with animated dots
- As SSE chunks arrive: append text to the AI message in real-time
- On "done" event: parse citations and display as badges
- Auto-scroll to bottom as new content arrives

### State Management
- `useState` for: messages array, input value, isExpanded, isLoading
- `useRef` for: message container scroll, abort controller
- No external state library needed

### Keyboard
- Enter to send, Shift+Enter for newline
- Escape to collapse

### Responsive
- On mobile: full-width bar, expanded state takes ~80vh
- On desktop: full-width bar, expanded state 60vh

## 9. Constraints

- Use ONLY Gemini File Search for RAG — no other vector DB or search tool
- File Search cannot be combined with Google Search, URL Context, or Code Execution tools simultaneously
- Gemini File Search Store data persists until manually deleted (no TTL)
- Max file size: 100MB per document (our files are well under this)
- Read Next.js 16 docs before creating API routes (per AGENTS.md)
- Do not commit API keys or `.env.local`

## 10. Success Criteria

1. User clicks the chat bar, types "What was the highest token usage day?" and gets an accurate answer with the correct date and token count
2. AI cites the source file (e.g., "cursor-usage.csv") in its response
3. Responses stream progressively (not all-at-once)
4. Chat UI matches the dark theme with purple accents
5. No regressions to existing report pages
6. Setup script is idempotent and prints clear instructions
